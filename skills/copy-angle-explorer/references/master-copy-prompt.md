# 마스터 카피 프롬프트 (세션당 1회 — 모든 각도 에이전트 공통 선행)

**사용법**: 이 프롬프트는 세션에서 [블록 2] 제품 사실 스펙을 한 번만 확정한 뒤,
`prompts/angle-NN-*.md`(각 에이전트가 맡은 각도 하나) 앞에 그대로 붙여서 각 에이전트에게
전달한다. 각 에이전트의 입력은 **이 파일 + 자기 angle 파일 + 고정된 제품 스펙 +
`hero-template.md`** 뿐이다 — 다른 각도 파일은 읽지 않는다(FR-5, NFR-5, 오염 방지).
제품 사실 스펙은 30개 시안 전체에서 **한 글자도 다르지 않게** 고정된다(FR-3).

---

## 1. 최상위 규율 (NFR-1) — 이 문서에서 가장 중요한 한 줄

**근거·사실[] 밖의 수치·수상·고객사·기능을 절대 만들지 마라. 애매하면 빼라.**

이 제품의 1순위 규율이다(FR-11). 지어낸 이득 하나가 이 제품의 가장 큰 사고다.
헤드라인 한 줄, 서브카피 한 문장이 아무리 아쉬워도 근거·사실[] 에 없는 숫자·%·통화·
"1위/최초/유일/넘버원" 류·고객사명·기능명은 절대 새로 만들지 않는다. 확신이 없으면
그 표현을 빼고 더 약한(근거 안에서 말할 수 있는) 표현으로 대체한다. `scripts/copy_lint.py`
가 이 규율을 기계적으로도 검사하지만, 린트를 통과하는 것과 무관하게 이 규율이 먼저다.

---

## 2. 제품 사실 스펙 스키마 (30 시안 공유·불변)

되물어 확정한 값을 아래 스키마로 채운다. 이 스펙은 세션 내내 다시 쓰지 않는다.

```
제품명        : 문자열
한줄설명      : 문자열
타겟          : 문자열 (누구를 위한 것)
핵심이득[]    : 문자열 배열 (1개 이상)
근거·사실[]   : 문자열 배열 — 수치·고유명사·수상·고객사 등 "말해도 되는 사실"의 전체 집합
언어          : 기본 "ko"
톤힌트?       : 선택 (예: "전문적", "친근")
```

**근거·사실[] 이 copy-lint 의 화이트리스트다.** 여기 없는 정량 표현(숫자·%·통화·
고유명사·"1위/최초/유일" 류)이 시안에 등장하면 지어낸 것으로 간주되어 FAIL 한다.
애매한 이득·근거는 되묻고, 답을 모르거나 건너뛰면 그 각도군은 약하게 표시하되
근거·사실[] 에는 추가하지 않는다(FR-2).

같은 내용을 시안 폴더의 `spec.json` 으로도 고정해 둔다 — `copy_lint.py` 가 실제로
읽는 것은 이 JSON 이며, **키 이름 `facts` 와 `language` 는 하드 계약**이다
(`load_spec()` 이 이 두 키만 보고, 없으면 `facts: []` 로 기본값을 채워 사실상 모든
정량 표현을 FAIL 시킨다 — 키를 다른 이름으로 적으면 화이트리스트가 빈 것과 같다).

```json
{
  "product_name": "제품명",
  "one_liner": "한줄설명",
  "target": "타겟 설명",
  "benefits": ["핵심이득 1", "핵심이득 2"],
  "facts": ["근거·사실 1 (예: 가입 고객 10000명)", "근거·사실 2 (예: 무료 체험 14일)"],
  "language": "ko",
  "tone_hint": "전문적"
}
```

## 3. 히어로 HTML 기술 요건

`references/hero-template.md` 를 그대로 쓴다 — 골격을 새로 짜지 않는다.

- `cx-headline`(`<h1>` 정확히 1개) / `cx-sub`(`<p>` 정확히 1개) / `cx-cta`(`<a>` 정확히
  1개) 클래스와 태그 구조를 그대로 준수한다 — `copy_lint.py` 는 이 세 클래스만
  정규식으로 파싱하므로, 클래스명이나 태그를 바꾸면 린트가 아무것도 못 읽는다.
- 헤드라인 **≤ 60자**, 서브카피 **≤ 140자**, CTA **정확히 1개**(결정5). 히어로는
  스크롤 없이 1 뷰포트 안에서 완결한다.
- 결과물은 **자기완결형 단일 `.html`** 이다 — 외부 런타임 의존 없이 그 파일 하나로
  열린다. 폰트는 hero-template.md 에 이미 박힌 Pretendard CDN 한 줄만 허용한다(NFR-3).
- JS는 연출 전용이다 — 꺼지거나 실패해도 카피는 온전히 읽혀야 한다. 새 JS 기능을
  추가하지 말고 hero-template.md 의 `prefers-reduced-motion` 처리를 그대로 둔다.
- 하단 푸터에 `ANGLE {n} · {각도명} · {패밀리}` 를 표기한다(FR-9). `{n}`은 각도
  번호(01~30), `{각도명}`/`{패밀리}`는 자기 angle 파일에 적힌 값 그대로 쓴다.

## 4. 중립 스타일 스펙 (모든 시안 공통 — 결정4)

색·타이포·여백은 hero-template.md 에 이미 값으로 고정돼 있다. 각도별로 이 값을
바꾸지 않는다 — 바꾸는 순간 "메시지 비교"가 "스타일 비교"로 오염된다(FR-8).

- **색**: 모노크롬(`--ink #14171a` / `--sub-ink #52585f` / `--bg #fafafa` /
  `--surface #ffffff` / `--border #e6e8eb`) + 단일 액센트(`--accent #2554f0`).
  각도마다 "이 각도엔 이 색이 어울린다"고 새 색을 넣지 않는다 — 액센트는 전체
  30개에서 하나다.
- **타이포**: Pretendard 한 폰트만 쓴다. 헤드라인/서브/CTA 각각의 크기·굵기·행간은
  hero-template.md 의 `.cx-headline` / `.cx-sub` / `.cx-cta` 규칙을 그대로 따른다.
- **여백**: 히어로는 뷰포트 중앙 정렬, `.cx-hero` 의 `padding`/`gap` 값을 그대로 쓴다.
  레이아웃을 좌측 정렬·2단 분할 등으로 바꾸지 않는다 — 배치도 스타일 변수다.
- 유일한 재량은 카피 텍스트 자체와(있다면) `.cx-trust` 보조 마이크로카피 문구뿐이다.

---

## 5. 언어

출력 카피는 제품 사실 스펙의 `언어`(기본 `"ko"`)를 따른다(NFR-4). 언어를 바꿔
달라는 지시가 없는 한 한국어로 쓴다. **실존 인물·실존 기업명은 예시로도 쓰지
않는다** — 사회적 증명·비교 각도(패밀리 C·B)에서 예시가 필요하면 가상의 이름을
쓰거나, 근거·사실[] 에 실제로 있는 이름만 쓴다. 근거·사실[] 에 없는 실존 고객사/
경쟁사 이름을 "예시일 뿐"이라며 넣는 것도 NFR-1 위반이다.
