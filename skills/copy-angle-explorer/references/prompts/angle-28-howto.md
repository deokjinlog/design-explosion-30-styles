# ANGLE 28 — How-to (패밀리 F: 구조·수사)

## 안내
- 이 파일만 읽고 작업을 완성하세요. 다른 각도 파일(angle-01 ~ angle-27, angle-29, angle-30)은 읽지 않습니다.
- 작업 시작 전 `master-copy-prompt.md`의 최상위 규율(진실성 게이트, 스펙 화이트리스트, 히어로 제약)을 먼저 따릅니다. 이 파일은 그 규율 위에 얹는 각도별 세부 지침입니다.
- 패밀리 F(구조·수사)는 **내용이 아니라 말의 형식**이 변수입니다. 이 각도는 같은 사실을 "절차형 안내"라는 형식에 담는다.

## 각도명
How-to — `~하는 법`이라는 절차형 문법으로 실용적 단계를 약속하는 각도. 독자는 결과보다 "실행 가능해 보이는가"에 먼저 설득된다. 단계가 명확할수록 심리적 진입 장벽이 낮아진다.

## 설득 구조
1. "~하는 법"이라는 절차형 프레임으로 카피를 연다 — 결과가 아니라 과정을 약속한다.
2. 실제로 존재하는 단계·기능을 근거로 절차를 제시한다 (지어낸 단계 금지).
3. 각 단계가 부담 없어 보이게 압축해 제시하되, 없는 소요시간이나 배수로 부담을 축소하지 않는다.
4. 마지막 단계 뒤에 CTA로 "지금 그 절차를 시작하라"는 행동을 잇는다.

## 헤드라인 공식
- `{결과}까지, {N}단계로 끝내는 법`
- `{작업} 하는 법: {핵심 동사구}만 하면 끝`
- `{도구/절차} 없이 {결과} 만드는 법`
- `{특정 대상}을 위한 {작업} 시작하는 법`

## 톤
친절한 안내자 톤. 명령형·청유형 어미(-하세요, -해보세요)를 사용한다. 단계 번호나 순서를 명시해 절차가 눈에 보이게 한다. 과장 없이 담백하게, 그러나 실행 가능성을 분명히 전달한다.

## 좋은 예
(예시 스펙 가정: 노트 앱 '플로우노트' — 태그 기반 정리, 링크 공유, 소요시간·배수 스펙 없음)
- "메모 정리하는 법: 폴더 대신 태그만 붙이면 끝" → 실제 태그 기능에 근거한 절차, 시간·배수 언급 없음.
- "회의록 공유하는 법: 링크 하나면 끝" → 실제 링크 공유 기능을 그대로 절차로 압축했을 뿐, 소요시간을 지어내지 않았다.

## 나쁜 예
- "3분 만에 끝내는 회의록 정리법" → 스펙에 없는 소요시간("3분")을 지어낸 사례. copy-lint 진실성 게이트 위반.
- "기존 방식보다 10배 빠른 정리법" → 스펙에 없는 배수("10배")를 지어낸 사례.
- "단 1단계로 끝!" (실제로는 여러 조작이 필요한데 단계 수를 축소) → 절차를 실제보다 적게 표기해 진입장벽을 속이는 것도 진실성 위반이다.

## 금지
- 스펙에 없는 소요시간 표기 금지 ("3분 만에", "하루 만에", "즉시" 등 근거 없는 시간 단정).
- 스펙에 없는 배수·퍼센트 비교 금지 ("10배 빠른", "50% 단축" 등).
- 실제로 존재하지 않는 단계를 있는 것처럼 나열하거나, 반대로 실제 단계 수를 줄여 표기해 부담을 축소하지 않는다.
- 각 단계는 실제 제품 기능과 1:1로 대응해야 하며, 절차처럼 보이려고 꾸며낸 순서를 만들지 않는다.
