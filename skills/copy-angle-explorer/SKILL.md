# Copy Angle Explorer

제품 한 줄을 받아 **동일한 제품 사실을 30가지 설득 각도**로 표현한 히어로 카피 시안을 만들고,
비교 → 조합 → 확정 → 메시지 지침 역추출까지 진행하는 워크플로. design-explosion-30-styles 의
짝 스킬 — 저쪽이 "콘텐츠 고정, 스타일 변수"라면 이쪽은 **"제품 사실 고정, 표현 각도만 변수"**다.

핵심 원칙: **제품 사실은 고정, 표현 각도만 변수.** 시안마다 숫자·기능·타겟이 달라지면 각도 비교가
아니라 각기 다른 제품을 비교하는 꼴이 된다. 그리고 그보다 위에 있는 최상위 규율 하나 —
**제품 사실 스펙 밖의 것은 어떤 각도에서도 지어내지 않는다.** 각도는 표현 방식이지 사실 창작 면허가
아니다.

## 워크플로 (7 스테이지)

### 1단계 — 입력 접수 · 되물음 · 제품 사실 스펙 확정

사용자가 준 제품 한 줄을 받는다. 부족하면 **부족한 것만** 되묻는다 — 타겟(누구를 위한 것인지),
핵심이득(1개 이상), 근거·사실(숫자·고객사·수상 등 "말해도 되는 사실"의 전체 집합) 중 이미 채워진
항목은 다시 묻지 않는다. 사용자가 "알아서 해줘"면 있는 정보로 최대한 채우고 요약에서 확인만 받는다.

확정된 스펙은 **`spec.json`으로 고정**한다 — 이 파일이 30개 시안 전체가 공유하는 유일한 사실
원천이자, 3단계 `copy_lint.py` 게이트의 화이트리스트다. 스키마:

```json
{
  "product": "제품명",
  "one_liner": "한줄설명",
  "target": "타겟",
  "benefits": ["핵심이득1", "핵심이득2"],
  "facts": ["말해도 되는 사실1 (숫자·고유명사·수상·고객사 등)"],
  "language": "ko",
  "tone_hint": "선택 — 예: 전문적/친근"
}
```

`facts[]`에 없는 정량 표현(숫자·%·통화·"1위/최초/유일" 류)은 이후 어느 단계에서도 등장하면 안 된다
(truthfulness.md §(a)).

### 2단계 — 30 각도 병렬 생성

한 번에 30개를 흩뿌리지 말고 **6개 패밀리 × 5개 각도**로 구조를 유지한 채 생성한다.

**병렬 모드 (서브에이전트 가능 환경 — Claude Code / Cowork / Workflow 도구):**
`spec.json`을 공유(고정)한 뒤 **각도당 1 에이전트를 병렬 실행**한다. **각 에이전트는 자기
`angle-NN-*.md` 파일 + `master-copy-prompt.md` + `hero-template.md` 만 읽는다 — 다른 각도
파일을 읽으면 그 순간 각도가 오염되고 30개가 평균화된 카피로 수렴한다.** 서브에이전트 없는 순수
채팅 환경이면 **패밀리 배치 직렬**로 폴백한다(패밀리당 5개씩 6배치). 환경에 따라 병렬/직렬이
자동으로 갈리되, 어느 쪽이든 30개가 모두 나온다.

각 에이전트의 입력은 정확히 세 가지다: ① `references/master-copy-prompt.md` (세션당 1회 —
제품 스펙 반영 규칙 + 히어로 기술요건 + "지어내지 않기" 최상위 규율) ② `references/prompts/angle-NN-*.md`
(해당 각도의 완성형 프롬프트 — 각도명·패밀리·설득 구조·헤드라인 공식·톤·좋은 예/나쁜 예·금지·크리틱
루프) ③ `references/hero-template.md` (모든 시안이 공유하는 중립 히어로 HTML 골격). 세 파일을
결합한 것이 그 시안의 실행 지시다.

파일명: `angle-NN-<slug>.html` 로 출력 디렉토리(`docs/copy-gallery/<날짜>-<제품슬러그>/`)에 저장한다.

### 3단계 — `copy_lint.py` 게이트 (하이브리드, 필수)

각 시안 저장 직후 다음을 실행해 히어로 제약과 진실성 규율을 결정론적으로 검사한다.

```
python3 skills/copy-angle-explorer/scripts/copy_lint.py <출력폴더>
```

같은 폴더의 `spec.json`을 화이트리스트로 자동 로드한다(폴더 인자를 넘기면 `gallery.html`을
제외한 `*.html` 전부를 훑는다). 판정 구성(truthfulness.md와 공유):

- **FAIL(기계적)**: 헤드라인 60자·서브 140자 초과, CTA ≠ 1개
- **FAIL(정량 클레임)**: `spec.json`의 `facts[]`에 없는 숫자·%·통화·최상급(1위·1등·최초·유일·최고·
  넘버원·업계 최고·세계 최초)
- **WARN**: 라틴 고유명사 — 오탐 여지가 있어 사람이 확인 (오탐이면 무시, 실제 위반이면 FAIL로 승격)
- **[MANUAL]**: 미묘한 진실성(과장·함의)·각도 구별성(표면 문구만 다르고 실제 설득 각도가 같은 경우) —
  판정 서브에이전트가 별도로 확인해야 하는 큐

**생성 → 린트 → 수정 → 재린트** 루프를 돈다. FAIL이 나오면 해당 `angle-NN-*.md`를 다시 읽고
위반을 수정한 뒤 재린트한다. **통과분만 사용자에게 제시**한다. 병렬로 뽑을수록 사람이 일일이
검수하기 어렵기 때문에 이 자동 게이트가 진실성의 최종 방어선이다.

### 4단계 — `gallery.html` 자동 생성

린트를 통과한 시안으로 **`gallery.html`** 을 자동 생성한다 — 꾸미지 말고 시안이 한눈에 보이게:

- **패밀리별로 그룹화한 카드 그리드**(A~F, 6묶음) + 각 카드에 `<iframe>` 미리보기 + 새 탭 링크
- self-contained 인라인 CSS, 반응형, 각 카드에 각도 번호·각도명·패밀리 표기

경로를 안내하고 5단계로 넘어간다.

### 5단계 — 퍼널 좁히기

30-way 비교를 3번의 5-way 비교로 접는다 (decision-framework.md §3).

- **1라운드 — 방향**: 6개 패밀리 대표 각 1개, 총 6개를 3초 컷으로 훑는다. **탈락 먼저** — 방향
  1~2개(패밀리 1~2개)를 남긴다.
- **2라운드 — 정독**: 남긴 패밀리 안에서 5개를 골라 헤드라인·서브·CTA까지 문장 단위로 정독한다.
- **3라운드 — 결승**: 2~3개를 나란히 놓고 실제 히어로 맥락에서 비교한다.

한 라운드에 5개를 넘기지 않는다 — 넘기는 순간 다시 평평한 30개로 돌아간다.

### 6단계 — 조합 하이브리드

결승까지 온 각도 중 완벽한 승자는 드물다. **축(헤드라인 / 서브 / CTA / 톤)별로 쪼개서 병합**한다.
예: "3번 각도 헤드라인 + 8번 각도 CTA + 12번 각도 톤". 하나를 통째로 고르려 하지 말고, 각 축에서
제일 강한 문장을 뽑아 붙인 하이브리드 시안을 만든다 (decision-framework.md §4).

### 7단계 — 확정 각도 역추출 + design-explosion 핸드오프

각도가 확정되면 그 결과를 **재사용 가능한 메시지 지침**으로 역추출한다:

1. **포지셔닝 한 줄** (이 제품을 한 문장으로 어떻게 부르는가)
2. **핵심 약속** (고객이 얻는 것 — 과장 없이 `spec.json`의 사실에 근거)
3. **톤 규칙** (문장 길이, 어휘 수준, 유머 허용 여부, 금지어)
4. **쓸 곳** (랜딩 히어로 / 광고 카피 / 이메일 제목 등 어디에 적용할지)

확정 각도(포지셔닝 + 톤 규칙)를 **design-explosion-30-styles 쪽으로 수동 핸드오프**한다 — 카피
축(이 스킬)과 스타일 축은 독립적으로 탐색하되 최종 결과물은 둘의 곱이다. **스타일 × 메시지 =
랜딩 완성.** 순서는 메시지 확정이 먼저다 — 카피가 흔들리는 상태에서 스타일을 입히면 스타일 비교의
"한 변수 고정" 원칙마저 오염된다.

### 빈 칸 리포트 — 카피 결과가 아니라 포지셔닝 진단

30개 각도를 다 뽑고 나면, **근거 부족으로 유독 약하게 나온 각도들을 패밀리별로 집계**해
사용자에게 되돌려준다 (decision-framework.md §7). 예: 사회적 증명 패밀리(후기·사용자 수·수상
인용) 6개가 전부 힘없이 뭉뚱그려졌다면 — 카피 실력 문제가 아니라 **제품에 아직 후기도, 사용자
수도, 인용할 근거도 하나도 없다는 뜻**이다. 카피는 없는 사실을 못 만든다. 이건 "어떤 카피가
이겼나"가 아니라 "우리 제품에 무엇이 없는가"의 질문 — 즉 카피 산출물이 아니라 **포지셔닝 진단**
이다.

## 공통 기술 요건 (모든 시안 공통)

- 단일 `.html` 파일, 순수 HTML/CSS/JS, self-contained — JS가 실패해도 HTML만으로 완전한 화면
- 한글 폰트: Pretendard CDN
  `https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.min.css`
- **`cx-*` 클래스 계약** (copy_lint.py 파싱 계약 — 이름을 바꾸면 린트가 카피를 못 읽는다):
  - `cx-headline` — 헤드라인, **60자 이하**
  - `cx-sub` — 서브카피, **140자 이하**
  - `cx-cta` — CTA 버튼, **시안당 정확히 1개**
- 히어로는 **1뷰포트 완결** — 스크롤 없이 헤드라인·서브·CTA가 한 화면에 보여야 각도 간 비교가
  길이·구성 차이로 오염되지 않는다
- 스타일은 `hero-template.md`의 중립 골격 그대로 — 색·타이포·레이아웃을 각도별로 바꾸지 않는다
  (스타일 변수는 design-explosion의 몫)
- 파일 하단 푸터에 `ANGLE {번호} · {각도명} · {패밀리}` 표기
- 제품 사실은 1단계에서 확정한 `spec.json`과 한 글자도 다르지 않게

## 참조 파일

- `references/master-copy-prompt.md` — 제품 스펙 반영 규칙 + 히어로 HTML 기술요건 + "지어내지
  않기" 최상위 규율. 세션당 1회 읽는다.
- `references/angle-taxonomy.md` — 30 각도 × 6 패밀리 인덱스 표. 각도 이름·번호·패밀리 소속을
  한눈에 확인할 때 참조.
- `references/hero-template.md` — 모든 시안이 공유하는 중립 히어로 HTML 골격(`cx-headline` /
  `cx-sub` / `cx-cta` 계약 + 푸터 슬롯 정의). 2단계에서 각 에이전트가 반드시 읽는다.
- `references/truthfulness.md` — 지어내지 않기 규율 + 클레임 판정 기준. `copy_lint.py`와 판정
  서브에이전트가 공유하는 단일 기준 — FAIL/WARN/[MANUAL] 3단 판정 근거.
- `references/decision-framework.md` — 퍼널·조합·역추출·빈 칸 리포트 프레임워크(design-explosion
  결정 프레임워크를 카피 축으로 각색). 5~7단계에서 적용.
- `references/prompts/` — 각도별 완성형 프롬프트 30개(`angle-01..30-*.md`). 2단계 시안 생성
  직전 해당 파일을 반드시 읽고 원문 그대로 실행한다. 다른 각도 파일은 절대 읽지 않는다.
- `scripts/copy_lint.py` — 하이브리드 카피 린터(기계적 FAIL + 정량 클레임 추출). 사용법:
  `python3 skills/copy-angle-explorer/scripts/copy_lint.py <출력폴더>` (같은 폴더의 `spec.json`을
  화이트리스트로 사용). FAIL 수정 후 재린트.
