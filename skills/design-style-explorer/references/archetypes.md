# 화면 원형(Archetype) 라이브러리 — 도메인에 맞는 화면을 고르는 자리

이 파일은 **"이 제품의 대표 화면은 어떤 종류인가"** 를 정하는 곳이다. 1단계 콘텐츠 인터뷰에서 원형 하나를 고르고, 그 원형의 컴포넌트 목록으로 [콘텐츠 스펙]을 채운다.

## 왜 이게 필요한가 (이 파일이 생긴 이유)

이전 버전은 `master-prompt.md` 에 **대시보드 컴포넌트가 하드코딩**돼 있었다 (사이드바·KPI 4개·차트 2종·목표 진행률·세그먼트·테이블…). "대시보드가 어색한 도메인은 등가 치환하되 컴포넌트 수준을 유지" 라는 단서가 있었지만, 실제로는 **컴포넌트 목록 자체를 이식**하는 사고가 반복됐다.

실제 사고 사례 — 문학 스토리 서비스(연서)에 대시보드 원형을 이식한 결과:
- `들은 스토리 1,284,000편` ← 개인 독서 서비스에 있을 수 없는 수치
- `목표 달성률 65%` ← 소양을 KPI로 만듦 (제품 철학과 정반대)
- `축 구성비 도넛 차트` ← 문학 읽기에 파이차트
- 정작 제품의 정체성인 **대화**는 우측 곁다리 패널로 밀림

30개 시안 전부가 "대시보드 + 채팅 위젯"이 되어, **"이 스타일이 이 제품에 맞나"를 판단할 수 없는 갤러리**가 나왔다. 시안 품질(린트 30/30 통과)과 무관하게 **질문 자체가 틀린** 것이다.

> **교훈: 30개 시안에서 고정할 것은 "컴포넌트 목록"이 아니라 "디자인 챌린지"다.**
> 풍부함은 필요하지만, **풍부함의 종류는 도메인이 정한다.**

## 도메인 무관 불변 — 디자인 챌린지 (모든 원형이 반드시 만족)

원형이 무엇이든 아래 9가지는 화면에 있어야 한다. 이게 없으면 스타일이 안 갈린다.

| # | 챌린지 | 왜 필요한가 |
|---|---|---|
| 1 | **정체성 요소** — 3초 안에 "이게 무슨 제품인지" 드러나는 것 | 스타일 판단의 기준점 |
| 2 | **명확한 위계** — 주인공 1 / 보조 2~3 / 배경 | 위계 처리가 스타일 역량의 핵심 |
| 3 | **반복 단위** — 카드·행·말풍선·문단 중 하나가 리듬을 만듦 | 반복은 스타일 언어가 가장 드러나는 곳 |
| 4 | **상태 4종** — 색 + 텍스트/아이콘 병행 | 상태 표현은 스타일마다 완전히 다름 |
| 5 | **밀도 지점 1곳** — 조밀한 영역(표·목록·긴 본문) | 여백 철학이 드러남 |
| 6 | **긴 것 / 짧은 것 극단** | 레이아웃이 버티는지 |
| 7 | **결측 · 빈 상태** | 빈 화면 처리는 스타일의 성숙도 |
| 8 | **액션** — 주요 1 + 보조 1 | 버튼은 스타일 정체성의 축약 |
| 9 | **내비게이션 / 위치 감각** | 구조를 보여주는 방식 |

## 원형 고르는 법 — 단 하나의 질문

> **"이 화면에서 사용자가 하는 단 하나의 행동은 무엇인가?"**

| 답 | 원형 |
|---|---|
| 훑는다 · 감시한다 · 상태를 확인한다 | **A. 대시보드형** |
| 대화한다 · 묻고 답한다 | **B. 대화형** |
| 읽는다 · 몰입한다 | **C. 읽기형** |
| 고른다 · 둘러본다 · 비교한다 | **D. 컬렉션형** |
| 입력한다 · 신청한다 · 단계를 밟는다 | **E. 폼·마법사형** |
| 흘려본다 · 소비한다 | **F. 피드형** |
| 만든다 · 편집한다 | **G. 에디터·캔버스형** |
| 찾는다 · 좁혀간다 | **H. 검색·결과형** |

**두 개가 섞이면 주된 것 하나를 고르고 나머지는 보조로 넣는다.** (예: 연서 = 대화한다 + 읽는다 → **B 대화형** 주, **C 읽기형** 보조)

---

## A. 대시보드형 — 훑는다 / 감시한다

**대표 예**: 관제·분석·백오피스·정산·모니터링

**컴포넌트**: 사이드바(메뉴 6) · 상단바(검색·알림·아바타·주요/보조 버튼) · 기간 탭 + 필터 · 알림 배너 · KPI 4(값+증감+스파크라인) · 차트 2종(추이 라인 + 구성비 도넛) · 목표 진행률 · 세그먼트 컨트롤 · 상위 항목 미니 리스트 · 테이블(5열×6행, 정렬·상태 4종·행 액션) · 활동 피드 또는 빈 상태

**스타일을 가르는 표면**: 숫자 타이포 · 카드 경계 · 데이터 밀도 · 차트 색 운용 · 상태 배지

**견고성 스트레스**: 억/조 큰 숫자 · 음수 증감 · 아주 긴 셀 · 결측 `—`

---

## B. 대화형 — 대화한다

**대표 예**: AI 어시스턴트 · 상담 · 챗봇 · 인터뷰형 제품

**컴포넌트**: 대화 흐름(**화면의 주인공 — 곁다리 패널 금지**, 화자 구분된 말풍선 4~6마디) · 보기 버튼(선택지 2~3) · 입력창 + 보내기 + 상태(작성중/대기) · 시스템 안내 배너 · 응답 카드(대화 결과물 — 이 제품의 산출물) · 대화 목록/히스토리(사이드) · 상단 정체성 바 · 빈 상태(첫 진입) · 되돌아보기(지난 결과 모음)

**스타일을 가르는 표면**: **말풍선 형태·정렬·간격 리듬** · 화자 구분 방식 · 대기/타이핑 표현 · 응답 카드의 무게 · 입력창의 존재감

**견고성 스트레스**: 아주 긴 발화(줄바꿈) vs 한 단어 발화 · 연속 발화 · 응답 실패 · 대화 0건

> ⚠️ 이 원형에 **KPI·차트·목표 진행률을 넣지 마라.** 대화가 밀려나면 원형을 배신한 것이다.

---

## C. 읽기형 — 읽는다 / 몰입한다

**대표 예**: 아티클 · 스토리 · 문서 · 뉴스레터 · 전자책

**컴포넌트**: 본문(**긴 텍스트가 주인공** — 400~800자, 문단 3~5) · 제목 + 부제 + 메타(작성자·시간·소요시간) · 발췌/인용 블록 · 핵심 한 줄(요약·교훈) · 출처/원전 표기 · 태그·분류 배지 · 목차 또는 진행 표시 · 다음 읽을거리 · 액션(저장·공유) · 빈 상태

**스타일을 가르는 표면**: **본문 타이포 스케일·행간·측정폭(measure)** · 인용 처리 · 제목 위계 · 여백 리듬 · 읽기 편안함

**견고성 스트레스**: 아주 긴 제목 · 한 줄 요약 · 인용 안의 인용 · 작가 미상(`—`) · 원문 병기 · 문단 1개짜리 짧은 글

> ⚠️ 이 원형의 승부처는 **타이포그래피**다. 차트를 넣으면 그 비교가 망가진다.

---

## D. 컬렉션형 — 고른다 / 둘러본다

**대표 예**: 커머스 목록 · 갤러리 · 카탈로그 · 서재 · 템플릿 목록

**컴포넌트**: 아이템 그리드(카드 8~12, **썸네일 자리 포함**) · 필터 사이드바 또는 칩 · 정렬 드롭다운 · 검색 · 결과 수 표시 · 카드 내부(썸네일·제목·메타·배지·액션) · 선택/비교 상태 · 페이지네이션 또는 더보기 · 빈 상태(필터 결과 0)

**스타일을 가르는 표면**: **카드 형태·그림자·간격** · 썸네일 비율 · 배지 처리 · 그리드 리듬 · hover 반응

**견고성 스트레스**: 아주 긴 제목(2줄 말줄임) · 썸네일 없음 · 배지 3개 겹침 · 결과 0건

---

## E. 폼·마법사형 — 입력한다 / 신청한다

**대표 예**: 가입 · 신청 · 온보딩 · 결제 · 설정

**컴포넌트**: 단계 표시(스텝 4, 현재 강조) · 입력 필드 6~8(텍스트·선택·날짜·체크·라디오·파일) · 필드 상태(기본·포커스·에러·비활성·완료) · 도움말 텍스트 · 에러 요약 배너 · 요약 패널(입력 결과) · 주요/보조 액션 · 저장됨 표시

**스타일을 가르는 표면**: **입력 필드 형태·포커스 링·에러 표현** · 레이블 위치 · 단계 표시 방식 · 폼 밀도

**견고성 스트레스**: 에러 3개 동시 · 아주 긴 도움말 · 선택 안 함 · 필수 누락

---

## F. 피드형 — 흘려본다

**대표 예**: 타임라인 · SNS · 알림 센터 · 로그 스트림

**컴포넌트**: 무한 스크롤 항목 6~8(작성자·시간·본문·미디어 자리·반응) · 작성 입력 · 필터 탭 · 새 항목 알림 · 반응/액션 바 · 스레드·답글 접힘 · 구분자(날짜) · 빈 상태

**스타일을 가르는 표면**: 항목 구분 방식(선/여백/카드) · 아바타 처리 · 시간 표기 · 반응 아이콘 · 스크롤 리듬

**견고성 스트레스**: 아주 긴 본문(더보기) · 미디어 없음 · 반응 0 · 연속 같은 작성자

---

## G. 에디터·캔버스형 — 만든다 / 편집한다

**대표 예**: 문서 편집기 · 디자인 툴 · 코드 에디터 · 노트

**컴포넌트**: 캔버스/편집 영역(주인공) · 툴바(도구 6~8) · 속성 패널(우측) · 레이어/구조 트리(좌측) · 선택 상태 · 저장 상태 표시 · 실행취소/재실행 · 단축키 힌트 · 빈 캔버스 상태

**스타일을 가르는 표면**: 툴바 아이콘 처리 · 패널 경계 · 선택 하이라이트 · 캔버스 배경 · 조밀한 컨트롤

**견고성 스트레스**: 도구 많음 · 레이어 깊은 중첩 · 선택 없음 · 저장 실패

---

## H. 검색·결과형 — 찾는다 / 좁혀간다

**대표 예**: 검색 결과 · 문서 검색 · 지식베이스

**컴포넌트**: 검색바(주인공, 쿼리 입력됨) · 결과 항목 6~8(제목·발췌·하이라이트·출처·메타) · 패싯 필터(좌측, 3그룹) · 결과 수 + 소요 시간 · 정렬 · 관련 검색어 · 페이지네이션 · 결과 0 빈 상태 · 오타 교정 제안

**스타일을 가르는 표면**: **검색어 하이라이트 처리** · 발췌 타이포 · 결과 구분 · 패싯 밀도

**견고성 스트레스**: 아주 긴 발췌 · 하이라이트 여러 개 · 출처 결측 · 결과 0건

---

## 원형이 목록에 없을 때

가장 가까운 원형을 고르고 **컴포넌트를 도메인 언어로 바꿔 쓴다.** 새 원형을 추가할 만하면 이 파일에 같은 형식(대표 예 / 컴포넌트 / 스타일을 가르는 표면 / 견고성 스트레스)으로 추가한다.

**절대 하지 말 것: 안 맞는 원형을 골라놓고 컴포넌트를 억지로 이식하기.** 그게 이 파일이 생긴 이유다.
