# [콘텐츠 스펙 — 로그인/가입] · 세션 고정 · 원형 J 인증형

> 로그인 화면(Enrico Login 2위). rubric: 신뢰·간결 1순위. 스타일만 변수.
> J 필수 region: auth-form · brand. 권장: action · banner. ⛔ KPI·도넛·차트·표·목표 진행률 금지.
> 승부처: **단일 카드의 무게** — 로고·입력 필드·소셜 버튼·배경(여백/이미지/그라디언트).

## 화면 구성 — data-region 부착
### 1. 브랜드 `data-region="brand"`  로고 **모아북** + 태그라인 `읽은 책을 한곳에`
### 2. 에러 배너 `data-region="banner"` (경고)
`이메일 또는 비밀번호가 올바르지 않아요.`
### 3. 인증 폼 `data-region="auth-form"` (단일 카드)
- 이메일 `reader@moabook.` — **에러: 이메일 형식을 확인해 주세요**
- 비밀번호 `••••••••` + 표시 토글 (포커스 상태)
- `로그인 상태 유지` 체크박스 + `비밀번호 찾기` 링크
- 주요 버튼 `로그인`
- 구분선 `또는`
- 소셜 로그인 3개: `Google로 계속` · `Apple로 계속` · `카카오로 계속`
### 4. 보조 링크 (공통 `action`)
`아직 계정이 없으신가요? 가입하기`
### 5. 약관 문구 (하단, 아주 긴 문구 견고성)
`로그인하면 모아북의 서비스 이용약관과 개인정보 처리방침에 동의하는 것으로 간주되며, 마케팅 정보 수신은 설정에서 언제든 끌 수 있습니다.`

## 상태 4종 (필드/버튼 상태)
`기본` / `포커스`(비밀번호) / `에러`(이메일) / `비활성`(제출 중 로그인 버튼) — 색+텍스트/아이콘 병행.

## 견고성/규칙
에러 필드 + 에러 배너 · 아주 긴 약관 문구 · 소셜 버튼 3개 · 제출 중 비활성.
브랜드·이메일 가상(실존 서비스명은 소셜 버튼의 Google/Apple/카카오만 — 제공자 표기라 허용). 숫자·문구 한 글자도 안 바꿈.
배경을 어떻게 쓰는지(여백/이미지/그라디언트)가 스타일마다 확 갈린다.
