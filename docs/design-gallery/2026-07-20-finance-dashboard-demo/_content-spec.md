# [콘텐츠 스펙 — 금융 대시보드] · 세션 고정 · 원형 A 대시보드형

> 갭 도메인(수동 정의). 진짜 대시보드가 맞는 유일 케이스라 원형 A. rubric: 신뢰·밀도 1순위. 스타일만 변수.
> A 필수 region: kpi · chart-trend · table (여기선 대시보드가 정답이라 KPI·차트가 필수)

## 화면 구성 — data-region 부착
### 1. 사이드바 `data-region="nav"`  로고 **자산나침반** + 메뉴 6(대시보드 선택·거래·자산·리포트·알림·설정)
### 2. 상단바 `data-region="identity"`  검색 + 알림 벨(뱃지 4) + 아바타 + 주요 `이체` + 보조 `내보내기`
### 3. 기간 탭 + 필터  `1일 · 1주(선택) · 1개월 · 1년` + `계좌 ▾`
### 4. 알림 배너(info)  `이번 달 예산의 82%를 사용했어요.`
### 5. KPI 4개 `data-region="kpi"` (값+증감+스파크라인)
| 지표 | 값 | 증감 |
|---|---|---|
| 총자산 | ₩1,284,320,000 ← 큰 자릿수 | +2.4% |
| 이번 달 지출 | ₩4,182,000 | −8.1% ← 감소 |
| 순현금흐름 | ₩+1,920,000 | +12.4% |
| 신용점수 | 921점 | +3 |
### 6. 차트 2종
(a) `data-region="chart-trend"` 최근 10일 자산 추이(라인, 세로축 0부터, 8일째 피크)
(b) `data-region="chart-part"` 지출 구성 도넛 5항목: 주거 34% · 식비 22% · 교통 16% · 여가 16% · 기타 12%
### 7. 목표 진행률 `data-region="goal"`  `비상금 목표 ₩10,000,000 중 6,500,000 (65%)`
### 8. 최근 거래 테이블 `data-region="table"` (5열×6행, 상태 4종)
| 일시 | 내역 | 분류 | 금액 | 상태 |
|---|---|---|---|---|
| 07-20 09:12 | 스타** 강남점 | 식비 | −5,600 | 완료 |
| 07-20 08:40 | **정기예금 만기 자동연장 및 우대금리 재적용 처리 안내 거래** ← 긴 셀 | 이자 | +34,200 | 완료 |
| 07-19 22:03 | 월세 이체 | 주거 | −820,000 | 대기 |
| 07-19 14:20 | 급여 | 수입 | +3,900,000 | 완료 |
| 07-19 11:05 | 알 수 없는 해외결제 | — ← 결측 | −12,400 | 문제 |
| 07-18 19:33 | 편의점 | 식비 | −3,100 | 진행 |
상태 4종: `완료` / `진행` / `대기` / `문제`
### 9. 빈 상태(우측 위젯 하나) `data-region="empty"`  `아직 예약된 이체가 없어요` → `반복 이체를 설정하면 여기 표시돼요.` → `이체 예약`

## 견고성/규칙  큰 자릿수(12억)·음수 증감·아주 긴 셀·결측 `—`·문제 상태·빈 위젯. 실존 금융기관 금지(스타**·가상). 숫자·문구 한 글자도 안 바꿈.
