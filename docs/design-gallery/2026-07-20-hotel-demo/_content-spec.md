# [콘텐츠 스펙 — 호텔 예약] · 세션 고정 · 절대 변경 금지

> **화면 원형: D 컬렉션형** (domains.md `Travel > Hotel` 프리셋). 대표 화면 = **호텔 검색 결과**.
> 모든 시안이 글자 하나 다르지 않게 공유한다. 스타일만 변수다.

## rubric
- 브랜드 성격: 믿음직하고 산뜻함. 여행의 설렘은 있되 과하지 않게.
- 타겟: 날짜·도시 정하고 "어디 묵을지" 고르는 사람.
- 1순위 목적: **전환**(예약까지) > 신뢰 > 차별화.

## ⛔ 이 원형에 넣으면 안 되는 것 (archetype-lint FAIL)
**KPI 지표 카드 · 도넛/구성비 차트 · 목표 진행률.** 호텔 고르는 화면은 관제 대시보드가 아니다.

## 화면 구성 (모든 스타일 공통) — 의미 영역에 data-region 부착

### 1. 상단 정체성/검색 바  `data-region="identity"`
로고 **여기묵자** + 검색 요약 칩: `서울` · `9/12 – 9/14 (2박)` · `성인 2` + `검색 수정` 버튼

### 2. 좌측 필터  `data-region="filter"`
- 가격: `₩50,000 – ₩500,000+` 슬라이더
- 성급: `5성 · 4성 · 3성` 체크
- 편의: `조식 · 무료취소 · 수영장 · 와이파이`
- 상단 정렬 드롭다운: `추천순 ▾` (외: 낮은 가격순 · 평점순)

### 3. 결과 헤더  `data-region="results"`
`서울 호텔 128곳` + 정렬 `추천순 ▾`

### 4. 호텔 카드 그리드 (6장)  `data-region="collection"` / 각 카드 `data-region="item-card"`
각 카드: 썸네일 자리 · 호텔명 · 위치 · 평점(별+숫자) · 편의 배지 2 · 1박 요금 · 상태 배지 · `예약` 버튼
상태 4종(색+텍스트 병행): `예약가능` / `마감임박` / `무료취소` / `단독특가`

### 5. 페이지네이션  (공통 `action`)
`‹ 1 2 3 … 22 ›` + 보조 버튼 `지도로 보기`

### 6. 빈 상태  `data-region="empty"` (필터 결과 0건 시연)
헤드라인 `조건에 맞는 호텔이 없어요` → 설명 `가격 범위를 넓히거나 편의 조건을 줄여보세요.` → CTA `필터 초기화`

## 데이터 규칙
- 호텔·지명은 가상. 실존 브랜드 금지. 숫자·문구 한 글자도 안 바꿈.
- 컬렉션형 승부처는 **카드 형태·그림자·간격·썸네일 비율·배지**.
