# [콘텐츠 스펙 — 종합 OTA] · 세션 고정 · 절대 변경 금지

> **화면 원형: D 컬렉션형** (domains.md `Travel > General` 프리셋). 대표 화면 = **통합 검색 결과**.
> 호텔 데모와 구분점: **버티컬 탭(숙소/항공/패키지) + 할인율·리뷰수**. 모든 시안이 글자 하나 다르지 않게 공유.

## rubric
- 브랜드 성격: 활기차고 믿음직. "지금 예약하면 이득" 느낌.
- 타겟: 여러 숙소·가격을 **비교**해서 제일 나은 걸 고르려는 사람.
- 1순위 목적: **전환** > 활기 > 신뢰.

## ⛔ 이 원형에 넣으면 안 되는 것 (archetype-lint FAIL)
**KPI 지표 카드 · 도넛/구성비 차트 · 목표 진행률.** 검색 결과 화면이지 대시보드가 아니다.

## 화면 구성 (모든 스타일 공통) — 의미 영역에 data-region 부착

### 1. 상단 정체성 + 버티컬 탭  `data-region="identity"`
로고 **떠나요트립** + 버티컬 탭 4개(하나 선택): `숙소`(선택) · `항공` · `항공+숙소` · `렌터카`
검색 요약 칩: `서울` · `9/12 – 9/14 (2박)` · `성인 2` + `검색 수정` 버튼

### 2. 좌측 필터  `data-region="filter"`
- 가격: `₩50,000 – ₩500,000+` 슬라이더
- 등급: `5성 · 4성 · 3성` 체크
- 편의: `조식 · 무료취소 · 수영장 · 조용한객실`
- 특가: `오늘특가만 보기` 토글

### 3. 결과 헤더  `data-region="results"`
`서울 숙소 240곳` + 정렬 `추천순 ▾` (외: 낮은 가격순 · 평점순 · 할인율순)

### 4. 결과 카드 그리드 (6장)  `data-region="collection"` / 각 카드 `data-region="item-card"`
각 카드: 썸네일 자리 · 숙소명 · 위치 · 평점(별+숫자)+리뷰수 · 편의 배지 2 · **원가(취소선)→할인가** · 할인율 배지 · 상태 배지 · `객실 보기` 버튼
상태 4종(색+텍스트 병행): `예약가능` / `마감임박` / `무료취소` / `오늘특가`

### 5. 페이지네이션  (공통 `action`)
`‹ 1 2 3 … 40 ›` + 보조 버튼 `지도로 보기`

### 6. 빈 상태  `data-region="empty"`
헤드라인 `조건에 맞는 숙소가 없어요` → 설명 `가격 범위를 넓히거나 특가 필터를 꺼보세요.` → CTA `필터 초기화`

## 데이터 규칙
- 숙소·지명·브랜드 전부 가상. 실존 브랜드 금지. 숫자·문구 한 글자도 안 바꿈.
- 컬렉션형 승부처: 카드 형태·그림자·간격·썸네일 비율·**할인 배지 처리**.
