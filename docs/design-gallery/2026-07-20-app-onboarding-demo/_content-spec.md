# [콘텐츠 스펙 — 앱 온보딩] · 세션 고정 · 원형 I 온보딩형

> 앱 첫 실행 튜토리얼(Enrico Tutorial 3위). rubric: 설렘·명료 1순위. 스타일만 변수.
> I 필수 region: slides · progress. 권장: action · brand. ⛔ KPI·도넛·목표 진행률(대시보드) 금지.

## 화면 구성 — data-region 부착
### 1. 브랜드/상단 `data-region="brand"`  로고 **하루한장** + 우상단 `건너뛰기`
### 2. 슬라이드 `data-region="slides"` (3~4장, 현재=2번째)
현재 슬라이드(크게):
- 일러스트 자리 (큰 이미지 영역)
- 헤드라인 **하루 한 장, 오늘의 나를 기록해요**
- 설명 `사진 한 장과 한 줄이면 충분해요. 매일 쌓이면 나만의 일 년이 됩니다. 부담 없이, 그날의 기분 그대로 남겨보세요.` ← 아주 긴 설명(견고성)
(뒤 슬라이드 미리보기 흐릿하게: 1 `환영해요`, 3 `친구와 함께`, 4 `알림 설정`)
### 3. 진행 표시 `data-region="progress"`  점 4개 중 2번째 채워짐  `● ● ○ ○` (2 / 4)
### 4. 액션 `data-region="action"`  주요 `다음` + 보조 `이전`  (마지막 슬라이드에선 `시작하기`)
### 5. 권한 요청 안내 (4번째 슬라이드 문구, 미리보기)
`알림을 켜면 매일 저녁 기록 시간을 알려드려요. 나중에 바꿀 수 있어요.`
### 6. 빈/엣지 (1번째 슬라이드가 될 때 `이전` 비활성)
`이전` 버튼 비활성 상태 시연.

## 상태 4종 (진행 점 + 슬라이드)
`현재`(채워진 점·활성 슬라이드) / `완료`(지나온 점) / `예정`(빈 점) / `건너뜀`(스킵 시)

## 견고성/규칙
아주 긴 설명문(3~4줄 wrap) · 일러스트 없는 폴백 대비 · 슬라이드 1개일 때 진행점 · `이전` 비활성.
브랜드·문구 가상. 숫자·문구 한 글자도 안 바꿈. 승부처: **일러스트 자리 처리·진행 표시 형태·헤드라인 타이포·여백·건너뛰기 존재감**.
