# 기여 안내

design-explosion-30-styles 에 관심 가져주셔서 고맙습니다.

이 저장소는 **하나의 화면을 30가지 디자인 스타일로 펼쳐 나란히 비교**하게 해주는 Claude Code 플러그인입니다. 스타일마다 전용 프롬프트가 있고, 린트가 규칙 위반을 잡습니다.

아래 원칙을 지켜서 함께 다듬어 주세요.

## 기여할 수 있는 내용

- 새 스타일 추가 — 기존 30개와 확실히 구별되는 시각 언어
- 기존 스타일 개선 — 팔레트 · 타이포 · 시그니처를 더 뚜렷하게
- 린트 규칙 보강 — 스타일별 "반칙"을 더 정확히 잡도록
- 최악 콘텐츠 테스트 보강 — 더 험한 조건에서도 안 무너지게
- 버그 제보 — 갤러리가 깨지거나, 스타일끼리 비슷해지는 경우
- 문서 개선 — 표현 · 예시 · 링크

## 지켜야 할 원칙

### 콘텐츠는 고정, 시각 언어만 바꿉니다

30개가 다른 건 디자인뿐이어야 합니다. 콘텐츠(문구 · 데이터)를 바꿔서 차이를 만들지 말아 주세요.

### 스타일마다 각본이 있습니다

무작위 생성이 아닙니다. 새 스타일은 매력 축 · 팔레트(hex) · 타이포 · 시그니처 · 금지 목록까지 전용 프롬프트로 못 박아 주세요.

### 금지는 린트로 강제합니다

"이 스타일에선 이건 반칙"을 말로만 적지 말고, 코드가 잡게 해주세요.

### 최악 콘텐츠로 테스트합니다

예쁜 더미가 아니라 긴 제목 · 큰 숫자 · 결측 · 빈 상태에서도 안 무너지는지 확인해 주세요.

### 문체를 지켜 주세요

자연스러운 한국어 존댓말을 유지하고, 한자는 쓰지 말아 주세요.

## 새 스타일을 추가하려면

두 파일을 함께 건드려 주세요.

1. **전용 프롬프트** — `skills/design-style-explorer/references/prompts/style-NN-*.md`
   매력 축 · 팔레트(hex) · 타이포 · 시그니처 · 금지 목록 · 크리틱 루프를 담습니다.
2. **린트 규칙** — `skills/design-style-explorer/scripts/style-lint.py`
   그 스타일에서 금지할 패턴(예: 특정 그림자 · 그라디언트)을 FAIL 규칙으로 넣습니다.

새 스타일이 기존 30개 중 어느 것과도 겹치지 않는지 먼저 확인해 주세요.

## 기여 절차

1. 변경 목적을 한두 문장으로 정리합니다.
2. 같은 스타일이나 문제를 다룬 이슈가 있는지 찾아봅니다.
3. 프롬프트와 린트를 함께 고칩니다.
4. 실제로 뽑아보고, 최악 콘텐츠에서도 안 무너지는지 확인합니다.
5. 갤러리에서 다른 스타일과 확실히 구별되는지 눈으로 봅니다.

버그 제보 · 제안 · 새 스타일 기여는 [GitHub Issues](https://github.com/deokjinlog/design-explosion-30-styles/issues) 로 받습니다.

## 검토 기준

- 기존 30개와 확실히 구별되는가
- 콘텐츠는 그대로 두고 시각 언어만 바뀌었는가
- 린트가 그 스타일의 반칙을 잡는가
- 최악 콘텐츠에서도 레이아웃이 버티는가

함께 다듬어 주시는 모든 개선이, 더 많은 분들이 자기 화면의 방향을 눈으로 고르게 돕습니다.
